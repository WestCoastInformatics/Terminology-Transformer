Introduction

This is a sample application for demonstrating the West Coast Informatics Transformer application. The "transformer" is a terminology-enabled platform that can be configured to answer a variety of different kinds of questions by passing data elements through terminology-based algorithms to produce a result. Follow the link for more information about the platform itself.

The NDC-RXNORM transformer application provides an API and basic user interface for interacting with RXNORM concepts, NDC (National Drug Codes), and SPL (Standard Product Label) Set Ids. The tool is loaded with all versions of RXNORM dating back to June 2007 (the first appearance of NDC code data).

As an instance of the Transformer application, this pipeline takes terminology codes (as either RXCUI, NDC, or SPL SetId) and transforms them into an information model representing broader and more historical picture of those data elements. While this is a relatively simple use case, it demonstrates how one type of data processed through terminology-enabled algorithms can emerge as a richer and more useful type of data.


Features

  • Deployable locally (to run within an existing application stack or behind a firewall)
  • Configurable security architecture
  • Supports "unit" and "batch" requests
  • Includes admin tools for easy updating to latest RXNORM version
  • Individual code and batch forms of API calls

REST API

The REST API comprises six functions, four of which represent the core services and two of which are used to support the user interface. The full documentation for the API can be found by clicking the API link in the header . Following is a description of the core services.

  • Get NDC Info

    • Takes an NDC as a parameter and returns a document containing the current RXNORM concept, its active status, the current SPL Set Id, and (optionally) the RXNORM concept history. The history is only provided if the "history=true" parameter is used.

      Examples

      curl -H Authorization:guest https://ndc.terminology.tools/rxnorm/ndc/00143314501?history=true
      curl -H Authorization:guest https://ndc.terminology.tools/rxnorm/ndc/0143-3145-01?history=true
      curl -H Authorization:guest https://ndc.terminology.tools/rxnorm/ndc/00143314501
      curl -H Authorization:guest https://ndc.terminology.tools/rxnorm/ndc/0143-3145-01
  • Get RXCUI Info

    • Takes an RXCUI (an RXNORM concept id) as a parameter and returns a document containing the RXNORM concept info, its active status, any associated SPL Set Ids, and (optionally) a complete history of NDCs associated with it. The history is only provided if the "history=true" parameter is used

      Example

      curl -H Authorization:guest https://ndc.terminology.tools/rxnorm/rxcui/351772?history=true
      curl -H Authorization:guest https://ndc.terminology.tools/rxnorm/rxcui/351772
  • Get NDC Properties

    • Takes an NDC as a parameter and returns a document a containing details of the NDC, including various properties associated with it in the current version of RXNORM.

      Examples

      curl -H Authorization:guest https://ndc.terminology.tools/rxnorm/ndc/00143314501/properties
      curl -H Authorization:guest https://ndc.terminology.tools/rxnorm/ndc/0143-3145-01/properties
  • Get NDC Properties for SPL Set Id

    • Takes an SPL Set Id as a parameter NDC as a parameter and returns a document containing details of all of the NDCs associated with it in the current version of RXNORM.

      Example

      curl -H Authorization:guest https://ndc.terminology.tools/rxnorm/spl/8d24bacb-feff-4c6a-b8df-625e1435387a/ndc/properties

Sample Invocations

Following are examples of calls that demonstrate the different ways of performing searches in this application.

  • Searching by name of a medication, e.g. "Aspirin"
  • Searching by RXNORM concept, e.g. "351772"
  • Searching by NDC, e.g. "00143314501"
  • Searching by SPL Set Id, e.g. "8d24bacb-feff-4c6a-b8df-625e1435387a"
